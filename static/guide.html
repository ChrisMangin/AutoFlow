What is AutoFlow?

AutoFlow is a desktop automation tool that lets you record mouse & keyboard actions, edit them into reliable workflows, and play them back hands-free. No scripting required for most tasks — just record, review, and run.

Workflows are stored as plain JSON files so they are portable and version-controllable. AutoFlow runs entirely locally — nothing leaves your machine.

ℹ️

AutoFlow lives in your system tray when the main window is closed. Double-click the tray icon or right-click it to reopen, record a new workflow, or replay the last one.

Recording

Recording captures every click, keystroke, and scroll you make across your entire desktop — including inside desktop apps, browsers, file explorers, and terminals.

Starting recording

  • Press F9 from anywhere — no need to click back to the browser first
  • Or click ● Record in the toolbar
  • Or click ● Rec on the HUD overlay

After starting, AutoFlow waits 1.5 seconds before capturing — use that time to switch to your target application. The first step is auto-detected: if you are in a browser, it records the current URL as a Navigate step; for other apps it records the executable path as an Open step.

💡

New Tab detection: AutoFlow now retries the start-context check up to four times, 400 ms apart. If you open a new browser tab and navigate to a page right as recording starts, it waits until it sees a real URL — not "New Tab" or an internal browser page.

HUD overlay

A frameless floating panel appears in the bottom-right corner at all times. Drag it anywhere by the title bar — position is remembered between sessions.

  • Idle: ● Rec   ▶ Play   ⏭ Step   ⏹ Stop
  • Recording: step count   ⏸ Pause   ⏹ Stop
  • Paused: ▶ Resume   ⏹ Stop
  • Playback: progress   ⏸ Pause   ⏭ Step   ⏹ Stop

Overlay buttons now send commands directly to the recorder — bypassing HTTP for reliability. Click Stop or Pause on the overlay and it responds instantly even while the hook thread is active.

What gets captured

  • Mouse clicks — with element name (accessibility tree), AutomationId fallback, region screenshot, and full-screen screenshot
  • Keyboard typing — batched into a single Type step per burst
  • Hotkeys — Ctrl+C, Alt+F4, function keys, etc.
  • Scroll wheel — consecutive scrolls at the same position merge automatically

Element detection

AutoFlow uses Windows UI Automation (UIA) to identify the element under each click. Resolution order:

  1. Accessible name (aria-label equivalent)
  2. Help text / tooltip
  3. AutomationId (developer-assigned identifier, e.g. submit-btn)
  4. Child walk — searches up to 4 levels deep in the accessibility tree for a named descendant
  5. Falls back to screen coordinates if nothing is found

The nearest named window ancestor (panel, pane, or window title) is also captured as context so playback can verify it is targeting the correct area.

⚠️

Chrome element names missing? Chrome's accessibility layer is sandboxed. Add a 🚀 Launch Browser step as step 1 to open Chrome with accessibility enabled, or start Chrome manually with --force-renderer-accessibility. For elements that still lack names, AutoFlow uses the AutomationId or the class name to give you context on the card badge.

Playback

Full playback

Click ▶ Play. AutoFlow replays every step from the start (or from your chosen start point). The active step highlights in the canvas as it runs.

Step-through mode

Click ⏭ Step to advance one step at a time. Each press executes exactly one step and pauses again — useful for debugging. You can start stepping from any point by clicking a step card first to mark it as the start point.

Start from a specific step

Click any step card body to mark it as the playback start — a green indicator appears at the card number. Both ▶ Play and ⏭ Step respect this start point.

Saving & Loading

Type a name in the title bar input and click 💾 Save (or Ctrl+S). Workflows are stored in %APPDATA%\AutoFlow\workflows\ as JSON files.

Click any workflow in the sidebar list to load it. To delete a workflow, click the ✕ next to it — the button turns red and shows Delete? for 3.5 seconds. Click again to confirm. If you don't click again, the delete cancels automatically.

The ● indicator in the page title and the pulsing Save button show unsaved changes.

Add Step Palette

The left sidebar contains collapsible sections grouping all step types. Click any step type chip to insert it at the end of your workflow (or after the selected step). Then click ✏️ Edit on the card to fill in the details.

Variables

Variables are named values stored in the right panel. Click ＋ Variable to add one. Give it a name (letters, numbers, underscores) and an optional default value.

Use variables anywhere a step accepts text: wrap the name in curly braces — {my_variable}. AutoFlow substitutes the live value at playback time.

Setting variables during playback

  • Set Variable — assign a literal value or expression
  • Get Clipboard — reads clipboard content
  • Read File — reads file content
  • HTTP Request — stores response body
  • Run Script — ctx["varname"] = value

Logic & Loops

Loop

The 🔁 Loop / ↩️ End Loop pair repeats everything between them N times.

🔁 Loop  (count: 5)
  🛵 Click  ...
  ⌨ Type   ...
↩ End Loop

If / Else

The ❓ If step branches on a variable value. Operators: == != > < contains.

❓ If  (status == "done")
  📢 Show Message: "Finished!"
↕️ Else
  📢 Show Message: "Still running..."
✓  End If

Error Handler

The 🛡️ Error Handler step wraps subsequent steps. If any step fails, you can choose to stop, skip the failed step, or retry (up to N times).

Run Script

The ⚙️ Run Script step executes an inline Python snippet. The ctx dict holds all variables — read from it, write to it:

# Double a numeric variable
ctx["count"] = int(ctx.get("count", 0)) * 2

# Build a dynamic filename
ctx["output"] = f"C:/reports/{ctx['date']}_report.txt"

Export & Import

PDF Report — 📄 PDF

Generates a printable HTML/PDF of the full workflow: step list with parameters, notes, and screenshots. Disabled steps appear with a "SKIPPED" tag.

Python Script — 🐍 Script

Exports a standalone .py file using pyautogui. Runs without AutoFlow installed. Good for scheduling via Windows Task Scheduler.

Runnable Package — 📦 ZIP

Exports a ZIP with the script, run.bat, requirements.txt, and a README. Recipients double-click run.bat — no AutoFlow required.

Import — 📥 Import

Accepts .json (AutoFlow workflow) or .py (exported Python script — round-trips back to editable steps).

Tips & Tricks

F9 global hotkey for hands-free recording

Press F9 at any time — even while your target app is in focus — to start recording instantly. Press F9 again to stop. The HUD overlay shows the step count as you record. You never need to click back to the AutoFlow browser window just to start or stop.

Step-through for debugging

Click a step card to mark it as the start, then press ⏭ Step repeatedly. Each press executes exactly one action and pauses. This lets you verify each step individually without running the full workflow.

Disabling steps

Click 🚫 on any step to disable it without deleting. Disabled steps are greyed out, skipped during playback, and commented out in exported scripts.

Undo deleted steps

Ctrl+Z restores up to 10 deleted steps in sequence, maintaining the original position.

Screenshot thumbnails

Each click step shows a region crop of the screen as a thumbnail. Click it to expand to the full screenshot in a lightbox. Thumbnails are stored as compact JPEG crops, not full-screen captures, so the workflow file stays small.

Start context detection

AutoFlow auto-detects the starting context when recording begins. It retries up to 4 times (400 ms apart) so it skips "New Tab" / blank pages and waits for the real URL after navigation. If no meaningful context is found after 1.5 s of retries, no start step is added.

Chrome web automation

  1. Add a 🚀 Launch Browser step as step 1
  2. Add a 🌐 Navigate step with your target URL
  3. Add the interaction steps

This opens Chrome with accessibility enabled so AutoFlow can identify elements by name rather than coordinates.

Variables for portability

Replace hardcoded paths and values with variables. Set them once with 📦 Set Variable, then all steps use {varname}. Changing targets means editing one field.

Multi-monitor recording

AutoFlow uses per-monitor DPI awareness. Coordinates are always physical pixels. The HUD overlay follows the primary monitor by default; drag it to any screen and its position is remembered. If playback clicks land in the wrong place, verify the window is on the same monitor as during recording.

Export script — supported steps

The Script and ZIP exports support: click, type, hotkey, scroll, wait, navigate, loop, if/else, set_variable, run_script, comment, open_file, kill_process, close_window, play_sound, read_file, write_file, copy_file, move_file, delete_file, http_request, wait_for_window, launch_browser, show_message, get/set_clipboard, screenshot.

Troubleshooting

Overlay Stop / Pause button does nothing

Fixed in the current version — overlay buttons now send commands through a direct in-process channel instead of HTTP, so they work instantly even while the recording hook is active. If you are running an older build, update to the latest autoflow.exe.

HUD stays on screen after closing the browser tab

Fixed — the HUD hides immediately when the last browser tab disconnects. A 300 ms grace window allows page refreshes to reconnect before the process exits, but the overlay itself disappears at the moment of disconnect.

Starting point says "New Tab"

Fixed — AutoFlow now retries the context detection up to 4 times, skipping blank/new-tab pages until a real URL appears. If you still get a wrong start step, simply delete it from the step list.

Playback clicks land in wrong positions

Coordinates are physical pixels (per-monitor DPI aware). Verify: (1) same monitor layout as recording, (2) target window same size/position, (3) no DPI change since recording.

No element names on step cards

AutoFlow tries accessible name, help text, AutomationId, and a 4-level child walk before giving up. If the badge still shows "element not detected", the element is inside a sandboxed renderer (Chrome content area). Add a 🚀 Launch Browser step as step 1, or start Chrome with --force-renderer-accessibility.

Screenshots not loading on step cards

Screenshots are captured ~80 ms after each click in a background thread. They appear shortly after the step card. If they never appear, the target application may have screenshot protection (video players, DRM content). On older builds, large full-screen screenshots were embedded in onclick attributes, causing DOM slowdown — this is fixed in the current build.

F9 hotkey not working

Another application may have registered F9 as a global hotkey. Try closing other automation tools or system utilities. AutoFlow registers F9 at startup; if the registration fails silently, you can still use the overlay buttons or the browser toolbar to start/stop recording.
